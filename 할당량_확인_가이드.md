# GCP 할당량 확인 가이드

## ✅ 계정 상태 확인

현재 상태:
- ✅ **계정이 유료로 업그레이드됨** (2025. 12. 6.)
- ⚠️ **Free Trial 크레딧이 남아있을 수 있음**
- ⚠️ **Free Trial 크레딧이 있으면 여전히 Free Trial 할당량 제한을 받을 수 있음**

## 🔍 할당량 확인 방법

### 1. Generative Language API 할당량 페이지 접속

1. **Google Cloud Console** 접속
   - https://console.cloud.google.com

2. **프로젝트 선택**
   - 상단에서 프로젝트 선택: `amazing-biz-blog`

3. **할당량 페이지로 이동**
   - 왼쪽 메뉴: **"API 및 서비스"** → **"할당량"**
   - 또는 직접 링크: https://console.cloud.google.com/apis/api/generativelanguage.googleapis.com/quotas

4. **필터 설정**
   - **서비스**: `Generative Language API` 선택
   - **메트릭**: `Requests per minute` 또는 `Tokens per minute` 선택

## 📊 Free Trial 제한 확인 방법

### 방법 1: 할당량 이름 확인

할당량 목록에서 다음을 확인:

**Free Trial 제한인 경우:**
- 할당량 이름에 **"Free Trial"** 또는 **"무료 체험판"**이 포함됨
- 예: "Free Trial: Requests per minute"
- 예: "무료 체험판: 분당 요청 수"

**유료 계정 제한인 경우:**
- 할당량 이름에 "Free Trial"이 없음
- 예: "Requests per minute"
- 예: "Tokens per minute"

### 방법 2: 할당량 값 확인

**Free Trial 제한 (매우 낮음):**
- **RPM (Requests Per Minute)**: 2-5 RPM
- **TPM (Tokens Per Minute)**: 10,000-50,000 TPM
- **일일 할당량**: 제한 있음

**유료 계정 제한 (높음):**
- **RPM (Requests Per Minute)**: 100+ RPM
- **TPM (Tokens Per Minute)**: 1,000,000+ TPM
- **일일 할당량**: 제한 없음 (사용량만큼 과금)

### 방법 3: 할당량 유형 확인

할당량 상세 페이지에서:
- **"할당량 유형"** 또는 **"Quota Type"** 확인
- **"Free Trial"** 또는 **"무료 체험판"** 표시 여부 확인

## ⚠️ Free Trial 크레딧이 남아있는 경우

### 문제점

**Free Trial 크레딧이 있으면:**
- ✅ 크레딧으로 비용이 상쇄됨
- ❌ **하지만 할당량 제한은 여전히 Free Trial 제한을 받음**
- ❌ **429 Too Many Requests 에러 발생 가능**

### 해결 방법

#### 방법 1: Free Trial 크레딧 사용 완료 대기
- 크레딧이 모두 소진될 때까지 대기
- 크레딧이 소진되면 자동으로 유료 계정 할당량으로 전환

#### 방법 2: Free Trial 크레딧 제거 (권장)
1. **크레딧 페이지 접속**
   - https://console.cloud.google.com/credits

2. **Free Trial 크레딧 확인**
   - 활성 상태인 "Free Trial" 크레딧 확인

3. **프로젝트에서 크레딧 제거**
   - 프로젝트 설정 → **"결제"** 메뉴
   - Free Trial 크레딧이 자동 적용되지 않도록 설정

#### 방법 3: 새 프로젝트 생성 (크레딧 없이)
1. **새 프로젝트 생성**
   - 크레딧 없이 생성

2. **유료 결제 계정 연결**
   - 새 프로젝트에 유료 결제 계정 연결

3. **API 키 재발급**
   - 새 프로젝트의 API 키 사용

## 💡 추가 팁

### 할당량 증가 요청

Free Trial 제한이 아닌데도 할당량이 부족한 경우:
1. **할당량 페이지**에서 **"할당량 편집"** 클릭
2. **"할당량 증가 요청"** 클릭
3. 필요한 할당량 값 입력
4. 요청 사유 작성
5. 제출

일반적으로 24-48시간 내에 승인됩니다.
