# 프롬프트 개선 제안

## 📋 현재 상태 분석

프롬프트를 검토한 결과, 대부분의 기능은 잘 구현되어 있지만 몇 가지 개선이 필요한 부분이 있습니다.

---

## ✅ 잘 구현된 부분

1. ✅ 판례 통합: D 섹션에 판례 관련 지침 포함됨
2. ✅ 담보별 보험료 필터링: B 섹션에 규칙 명시됨
3. ✅ 종합보험 주요 담보: B 섹션에 "5-7개" 규칙 있음
4. ✅ 톤 개선: "딱딱한 표현 금지" 명시됨
5. ✅ Google Custom Search: E 섹션에 검색 결과 활용 지침 있음

---

## 🔧 개선이 필요한 부분

### 1. **판례 활용 지침 강화** ⚠️ 중요

**현재 상태:**
- 판례가 제공되었을 때만 간단한 사용 규칙만 있음
- 판례를 본문에 **어떻게 자연스럽게 통합**할지 구체적 예시 부족

**제안:**
```
### D. [관련 분쟁조정 사례 - 반드시 활용하세요!]
...
**판례 활용 방법 (구체적 예시):**

1. **도입부 활용:**
   - ❌ "금융분쟁조정위원회 제2023-1234호에서..." (너무 딱딱함)
   - ✅ "실제로 최근 ${precedents[0].caseNumber} 사례에서 보듯이, 이런 상황이 종종 발생합니다..."
   - ✅ "고객님들이 많이 궁금해하시는 부분인데, ${precedents[0].caseNumber} 사례가 좋은 참고가 됩니다"

2. **본문 중간 활용:**
   - ✅ "이런 경우 실제로 어떻게 되는지 ${precedents[0].caseNumber}에서 확인할 수 있어요"
   - ✅ "혹시 이런 상황을 겪으셨나요? ${precedents[0].caseNumber}에서 다룬 사례와 유사할 수 있습니다"

3. **결론부 활용:**
   - ✅ "앞서 언급한 ${precedents[0].caseNumber} 사례처럼, 사전에 확인하는 것이 중요합니다"

4. **⚠️ 주의사항:**
   - 최대 1-2개 판례만 자연스럽게 포함
   - 판례 내용을 그대로 복사하지 말고, 본문 맥락에 맞게 재구성
   - 판례를 언급할 때마다 사건번호를 정확히 표기
```

---

### 2. **Google Custom Search 결과 활용 지침 강화** ⚠️ 중요

**현재 상태:**
- 검색 결과가 있으면 표시되지만, **구체적으로 어떻게 활용**할지 명확하지 않음
- 검색 결과의 통계/데이터를 본문에 포함하는 방법이 애매함

**제안:**
```
### 🔍 실시간 검색 결과 (최신 정보 참고)
...
**검색 결과 활용 지침:**

1. **통계 데이터 활용:**
   - 검색 결과에 통계가 있으면 → 본문에 자연스럽게 포함
   - 예: "최근 [검색 결과 출처]에 따르면..." 
   - ⚠️ 반드시 출처를 "참고 자료 및 출처" 섹션에 추가

2. **최신 동향 활용:**
   - 검색 결과의 최신 정보를 본문에 반영
   - 예: "2025년 기준으로..." (검색 결과 기반)

3. **출처 명시:**
   - 검색 결과를 사용했다면 → 반드시 출처 섹션에 URL 추가
   - Google Custom Search에서 제공한 URL 그대로 사용

4. **⚠️ 주의사항:**
   - 검색 결과가 주제와 관련이 없으면 무리하게 사용하지 않기
   - 검색 결과의 정보를 그대로 복사하지 말고 재구성
```

---

### 3. **담보별 보험료 표시 위치 명확화** ⚠️ 중요

**현재 상태:**
- 담보별 보험료 필터링 규칙은 있지만, **테이블에 어떻게 표시**할지 구체적 예시 부족

**제안:**
```
**⚠️ 매우 중요 - 담보별 보험료 표시 규칙:**

1. **테이블에 표시하는 경우:**
   - "담보별 보험료" 컬럼 추가
   - 각 담보를 줄바꿈(<br/>)으로 구분
   - 예: "암진단비 50,000원<br/>암수술비 40,000원<br/>암입원비 30,000원"
   - ⚠️ 주제와 관련 없는 담보는 제외

2. **본문에 설명하는 경우:**
   - 테이블은 총액만 표시
   - 본문에서 "A사의 총 보험료 136,216원은 암진단비 50,000원, 암수술비 40,000원, 암입원비 30,000원으로 구성됩니다" 형식으로 상세 설명

3. **종합보험의 경우:**
   - 주요 담보만 표시 (암, 뇌, 심장, 상해 등 5-7개)
   - 예: "암진단비 50,000원<br/>뇌졸중 진단비 40,000원<br/>상해사망 30,000원<br/>... 등"
```

### 4. **톤 개선 지침 구체화** ⚠️ 중요

**현재 상태:**
- "딱딱한 표현 금지"는 있지만, **어떤 표현을 사용하면 좋은지** 구체적 예시 부족

**제안:**
```
### B. Tone & Manner (문체 전략 - 매우 중요!)

**절대 금지:**
- ❌ "본 글에서는...", "이번 포스팅에서는..." 같은 형식적 표현
- ❌ "데이터 분석 결과", "검증 결과", "리포트" 같은 딱딱한 표현
- ❌ "AI가 분석한 결과", "자동 생성된" 같은 느낌의 표현
- ❌ "결론적으로...", "요약하면..." 같은 학술적 표현

**필수 지침 (구체적 예시):**

1. **자연스러운 대화체:**
   - ✅ "이거 정말 중요해요" (❌ "이는 매우 중요합니다")
   - ✅ "꼭 체크해보세요" (❌ "반드시 확인해야 합니다")
   - ✅ "저도 처음엔 잘 몰랐는데..." (❌ "초기에는 이해하기 어려웠습니다")

2. **경험담 스타일:**
   - ✅ "제가 실제로 고객님들 상담하면서 많이 받는 질문이에요"
   - ✅ "솔직히 말씀드리면, 이 부분은 정말 복잡해요"
   - ✅ "많은 분들이 이렇게 생각하시는데..."

3. **친근한 어조:**
   - ✅ "고객님이 궁금해하실 만한 부분이에요"
   - ✅ "혹시 이런 상황을 겪으신 적 있나요?"
   - ✅ "이건 제가 직접 경험한 케이스인데요"

4. **블로그/카페 스타일:**
   - ✅ "요즘 이런 질문 많이 받아요"
   - ✅ "오늘은 이거 한번 정리해볼게요"
   - ✅ "정말 많은 분들이 헷갈려하시는 부분이에요"
```

### 5. **제안서 분석 결과 활용 강화** 💡 개선

**현재 상태:**
- 제안서 분석 결과를 언급하지만, **어떤 부분을 우선적으로 활용**할지 명확하지 않음

**제안:**
```
### 📄 제안서 분석 결과 (참고 자료)

**우선 활용 순서:**

1. **고객 고민점 (worryPoint)**: 도입부나 본문 초반에 자연스럽게 통합
   - 예: "많은 고객님들이 [고민점]을 걱정하시는데요..."

2. **주요 장점 (sellingPoint)**: 본문 중간에서 장점 설명 시 활용
   - 예: "이런 보험이 특히 좋은 이유는 [주요 장점] 때문이에요"

3. **대상 고객 (targetPersona)**: 본문에서 타겟팅 시 활용
   - 예: "[대상 고객]에게 특히 추천드리고 싶은 이유는..."

4. **⚠️ 상품명은 절대 노출하지 않기!**
```

### 6. **출처 섹션에 판례/검색 결과 포함 규칙** 💡 추가

**현재 상태:**
- 출처 섹션 작성 규칙은 있지만, **판례나 검색 결과**를 언급하는 규칙이 없음

**제안:**
```
**출처 작성 체크리스트에 추가:**

- [ ] 판례를 본문에서 인용했다면 → "분쟁조정 사례: 금융분쟁조정위원회 [사건번호]" 추가
- [ ] Google Custom Search 결과를 사용했다면 → 검색 결과 URL 추가
- [ ] 판례와 검색 결과를 모두 사용했다면 → 둘 다 추가

**출처 섹션 예시 (판례 + 검색 결과 포함):**
```html
<div class="box" style="background: #f0f9ff; margin-top: 50px;">
  <p><strong>📚 참고 자료 및 출처</strong></p>
  <ul>
    <li>보험료 데이터: 자체 수집 데이터 (${todayStr} 기준)</li>
    ${precedents && precedents.length > 0 ? `<li>분쟁조정 사례: 금융분쟁조정위원회 ${precedents.map(p => p.caseNumber).join(', ')}</li>` : ''}
    ${searchResults ? `<li>최신 정보: [검색 결과 출처 URL들]</li>` : ''}
  </ul>
</div>
```
```

### 8. **다이어그램 다양성 지침 강화** 💡 개선

**현재 상태:**
- "매번 다른 스타일" 규칙은 있지만, **어떻게 랜덤하게 선택**할지 명확하지 않음

**제안:**
```
**다이어그램 선택 전략:**

1. **매번 생성 시마다 다른 카테고리에서 선택:**
   - 이전 생성: Bar Chart → 다음 생성: Pie Chart
   - 이전 생성: Venn Diagram → 다음 생성: Tree Diagram

2. **색상도 매번 변경:**
   - 이전: Blue/Navy → 다음: Purple/Orange
   - 이전: Soft Red → 다음: Green/Teal

3. **레이아웃도 다양하게:**
   - 이전: 수평 → 다음: 수직 또는 원형
```

## 📝 추가 제안

**현재 상태:**
- 판례가 없으면 분쟁사례를 언급하지 말라고만 되어 있음
- **대체 표현 예시** 부족

**제안:**
```
판례가 제공되지 않은 경우:

✅ 허용되는 표현:
- "실무에서 자주 접하는 케이스는..."
- "많은 고객님들이 이런 상황을 겪으시는데..."
- "보험업계에서 흔히 발생하는 사례를 보면..."
- "일부 분쟁 사례에서 확인되는 것은..."

❌ 절대 금지:
- "금융분쟁조정위원회 제2023-XXX호" (마스킹)

```

```

## 💬 결론

전반적으로 프롬프트는 잘 구성되어 있습니다. 다만 **구체적 예시와 활용 방법**을 더 명확하게 제시하면 AI가 더 정확하게 생성할 수 있을 것 같습니다.

특히 **판례와 검색 결과를 본문에 자연스럽게 통합하는 방법**을 구체적으로 명시하는 것이 가장 중요해 보입니다.
