# 할당량이 보이지 않는 경우 해결 방법

## 🔴 현재 상황

- 필터 적용: `requests per minute`, `tokens per minute`, `gemini-2.5-pro`
- 결과: **"지정한 옵션으로 제공되는 할당량이 없습니다."**

---

## 💡 해결 방법

### 방법 1: 필터를 하나씩 제거

1. **모든 필터 제거**
   - 각 필터 옆의 **"X"** 버튼 클릭하여 제거
   - 또는 필터 바의 **"모두 지우기"** 클릭

2. **전체 할당량 목록 확인**
   - 필터 없이 전체 1,137개 할당량 확인
   - 할당량 이름을 직접 확인

3. **필터를 하나씩 추가**
   - 먼저 `requests per minute`만 검색
   - 그 다음 `tokens per minute`만 검색
   - 마지막으로 `gemini-2.5-pro` 추가

### 방법 2: 다른 검색어 시도

할당량 이름이 다를 수 있으므로 다음 검색어를 시도:

### 방법 3: 할당량 유형 확인

1. **"유형" 컬럼 확인**
   - "할당량" (Quota)만 표시되는지 확인
   - "시스템 한도" (System Limit)도 확인

2. **필터에서 유형 선택**
   - "유형" 필터에서 "할당량" 선택
   - 또는 "시스템 한도" 선택

### 방법 4: 측정기준 확인

1. **"측정기준" 필터 확인**
   - 지역별로 할당량이 나뉘어 있을 수 있음
   - "측정기준" 필터에서 "전체" 또는 특정 지역 선택

2. **지역별 할당량 확인**
   - "asia-northeast1" (서울)
   - "asia-northeast2" (오사카)
   - "us-central1" 등

## 🔍 예상되는 할당량 이름

할당량 이름이 정확히 일치하지 않을 수 있으므로, 다음 패턴을 찾아보세요:

### Requests per minute
- "Requests per minute"
- "Request per minute"
- "RPM"
- "분당 요청 수"
- "Requests per minute per project"
- "Requests per minute per region"

### Tokens per minute
- "Tokens per minute"
- "Token per minute"
- "TPM"
- "분당 토큰 수"
- "Tokens per minute per project"
- "Tokens per minute per region"

### 모델별 할당량
할당량이 모델별로 나뉘어 있지 않을 수 있습니다:
- 모든 모델에 공통 적용되는 할당량일 수 있음
- 모델 이름이 할당량 이름에 포함되지 않을 수 있음

---

## 📋 단계별 확인 방법

### Step 1: 모든 필터 제거
1. 모든 필터의 **"X"** 버튼 클릭
2. 전체 할당량 목록 확인 (1,137개)

### Step 2: "request"만 검색
1. 검색 바에 `request` 입력
2. 결과 확인
3. 할당량 이름에 "per minute"가 포함되어 있는지 확인

### Step 3: "token"만 검색
1. 검색 바에 `token` 입력
2. 결과 확인
3. 할당량 이름에 "per minute"가 포함되어 있는지 확인

### Step 4: "gemini" 검색
1. 검색 바에 `gemini` 입력
2. 결과 확인
3. 모델별 할당량이 있는지 확인

### Step 5: 할당량 값 확인
찾은 할당량에서:
- **값 (Value)** 확인
- **현재 사용량** 확인
- **Free Trial 제한인지 확인** (값이 매우 낮으면 Free Trial)

---

## 🎯 Free Trial 제한 확인

할당량을 찾으면:

### Free Trial 제한인 경우
- **RPM**: 2-5
- **TPM**: 10,000-50,000
- 할당량 이름에 "Free Trial" 포함 (선택사항)

### 유료 계정 제한인 경우
- **RPM**: 100+
- **TPM**: 1,000,000+
- 할당량 이름에 "Free Trial" 없음

---

## ⚠️ 할당량이 없는 경우

만약 RPM/TPM 할당량이 전혀 보이지 않는다면:

### 가능성 1: 프로젝트에 할당량이 설정되지 않음
- 새 프로젝트일 수 있음
- API를 처음 사용하는 경우

### 가능성 2: 할당량 이름이 다름
- Google이 할당량 이름을 변경했을 수 있음
- 다른 형식으로 표시될 수 있음

### 가능성 3: Free Trial 크레딧으로 인한 제한
- Free Trial 크레딧이 있으면 할당량이 다르게 표시될 수 있음
- 크레딧 페이지에서 확인 필요

---

## 🔗 대안 확인 방법

### 1. API 사용량 확인
- "측정항목" (Metrics) 탭 확인
- 실제 사용량 확인
- 429 에러 발생 시점과 사용량 비교

### 2. 크레딧 페이지 확인
- https://console.cloud.google.com/credits
- Free Trial 크레딧 확인
- 크레딧이 있으면 할당량 제한 확인

### 3. 결제 계정 확인
- https://console.cloud.google.com/billing
- 결제 계정 상태 확인
- Free Trial인지 유료 계정인지 확인

---

## 💡 권장 사항

1. **모든 필터 제거 후 전체 목록 확인**
2. **"request" 또는 "token"만 검색**
3. **할당량 값이 매우 낮은지 확인** (Free Trial 제한)
4. **API 사용량과 429 에러 발생 시점 비교**

할당량이 보이지 않아도, **429 에러가 발생한다면 할당량 제한에 걸린 것**입니다.
